Human-Computer Interaction
Report

Summary

Before proceeding with this project, we created a prototype by writing an hci report. In order to make the application more efficient, we conducted some interviews to people around us. Moreover we created the expected users of our products and created a scenario for the two personas. We sketched the application's interface with our hands. After some feedbacks, the interface was implemented using a program called ​proto.io. To make it as intuitive as possible and easy to use, we went through a lot of feedback. After we completed the prototype, we made a report to record what we have done.

Our Goal

Useful

We tried to create an application that could be used most efficiently by the expected users we created.

Visibility

We tried to make it easier for users to see graphs and tables at a glance.

Easy to use

There must be some people who have difficulty using smartphones because it is a program that uses It devices. We hope people like that can use it easily.

Our First Prototype Sketches

After conducting the interviews, creating personas and writing scenarios, we have designed two versions of sketches, to visualize our application. By having two versions, we were able to present users with alternatives, which allowed them to better understand the design choices, that need to be taken, to create the application. It was a basic sketches for our project, so many feedbacks were needed.

Evaluation of the Prototype

Heuristic evaluation was used due to its lower time demand, ease of use and quick feedback. Moreover, the scenario based evaluation was performed, where a user was directed a task to be completed and time was measured.

Our Final Prototype

We fixed all the shortcomings and created the easiest final prototype to see and use.
